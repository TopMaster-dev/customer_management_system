# 顧客管理システム 利用ガイド

このガイドでは、顧客管理システムの**機能の概要**と、**プロジェクトの起動方法**を説明します。

---

## 1. プロジェクトの概要

本システムは、店舗・顧客・スタッフ・来店記録・売上・経費を一括で管理するWebアプリケーションです。

- **ブラウザ**で操作するため、PC・スマートフォンどちらからでも利用できます。
- ログインした**権限（ロール）**に応じて、使える機能が変わります。
- 管理者は全店舗のデータを扱え、それ以外の方は**自分が所属する店舗のデータのみ**閲覧・編集できます。

---

## 2. 機能一覧

### 2.1 認証・アカウント

| 機能 | 説明 |
|------|------|
| **ログイン** | メールアドレスとパスワードでログインします。 |
| **新規登録** | 初めての登録時は**管理者**として作成されます。すでに管理者がいる場合は**キャスト**として、店舗を選んで登録します。 |
| **マイページ** | 自分の目標・売上サンプルの確認、メールアドレス・パスワードの変更ができます。権限（ロール）の変更は管理者のみ可能です。 |

### 2.2 顧客管理

| 機能 | 説明 |
|------|------|
| **お客様一覧** | 登録済み顧客を検索・一覧表示します。名前・店舗・初回来店日などで絞り込みできます。 |
| **お客様登録** | 顧客の基本情報を登録します。登録後、詳細（プロフィール・詳細・嗜好）をモーダルから追加できます。 |
| **お客様の編集・削除** | 一覧や詳細から、顧客情報の編集・削除ができます。詳細情報も同じ画面から編集できます。 |

### 2.3 来店・売上・経費

| 機能 | 説明 |
|------|------|
| **来店記録** | お客様の来店日・利用額・支払方法・入退店時刻などを登録・編集・削除します。 |
| **日次売上** | 来店記録の利用額を店舗・日付ごとに集計して表示します。参照専用で、ここから直接の登録・編集はできません。 |
| **日次経費** | 店舗・日付ごとに、売上合計・経費・人件費・備考を登録・編集・削除します。売上合計は来店記録から自動計算されます。 |

### 2.4 店舗・ユーザー・スタッフ

| 機能 | 説明 |
|------|------|
| **店舗管理** | 店舗の登録・一覧・編集・削除。**新規店舗の作成は管理者のみ**可能です。それ以外の方は自店舗の表示・編集のみです。 |
| **ユーザー管理** | キャスト・スタッフ・マネージャー・管理者のアカウントを登録・編集・無効化します。店舗の割り当てもここで行います。管理者のみ全ユーザーを管理でき、それ以外は自店舗のユーザーなどに制限されます。 |
| **スタッフ管理** | どのユーザーをどの店舗のスタッフとして登録するかを管理します。店舗は自店舗のみ選択でき、キャストはその店舗に属するユーザーのみ選択できます。 |

### 2.5 権限（ロール）ごとの利用範囲

| ロール | 利用できる主な機能 |
|--------|----------------------|
| **キャスト** | ホーム、お客様一覧・登録、来店記録、マイページ |
| **スタッフ** | 上記 ＋ 日次売上、日次経費、スタッフ管理 |
| **マネージャー** | スタッフの範囲 ＋ 店舗管理（自店舗の表示・編集） |
| **管理者** | 上記すべて ＋ 店舗の新規作成、全ユーザー管理、全店舗のデータ操作 |

## 3. 動作環境

次の環境で動作を想定しています。

- **Python** 3.10 以上
- **Node.js** 18 以上
- **PostgreSQL** 14 以上（本番・推奨）。開発時は SQLite に切り替えることもできます。
- ブラウザは、Chrome・Firefox・Safari などの最新版を推奨します。

---

## 4. プロジェクトの起動方法

### 4.1 リポジトリの取得

```bash
git clone <リポジトリのURL>
cd customer_management_system
```

### 4.2 バックエンド（Django API）の準備

**1. 仮想環境の作成と有効化**

```bash
cd backend
python3 -m venv venv
```

- **Linux・macOS:** `source venv/bin/activate`
- **Windows:** `venv\Scripts\activate`

**2. パッケージのインストール**

```bash
pip install -r requirements.txt
```

**3. データベース（PostgreSQL）の準備**

PostgreSQL を使う場合、あらかじめデータベースとユーザーを作成してください。

```bash
sudo -u postgres psql
```

```sql
CREATE USER cms_user WITH PASSWORD 'ここにパスワードを設定';
CREATE DATABASE customer_management_db OWNER cms_user;
\c customer_management_db
GRANT USAGE, CREATE ON SCHEMA public TO cms_user;
\q
```

`backend/backend/settings.py` の `DATABASES` で、`PASSWORD` などを実際の設定に合わせて変更してください。

**4. マイグレーションの実行**

```bash
python manage.py migrate
```

**5. 開発サーバーの起動**

```bash
python manage.py runserver
```

バックエンドは **http://localhost:8000** で動作します。このターミナルは起動したままにしてください。

---

### 4.3 フロントエンド（React）の準備と起動

**別のターミナル**を開いて次を実行します。

```bash
cd frontend
npm install
npm start
```

ブラウザが開き、**http://localhost:3000** で画面が表示されます。API への通信は開発時は自動的にバックエンド（http://localhost:8000）に転送されます。

---

### 4.4 初回利用の流れ

1. ブラウザで **http://localhost:3000** を開く。
2. まだユーザーがいない場合は **「新規登録」** から最初のユーザーを登録する。このユーザーは**管理者**になります。
3. 店舗がまだない場合でも、管理者は店舗なしで登録できます。ログイン後、**店舗管理**から店舗を登録してください。
4. ログイン後、画面上部のメニューから各機能に移動できます。
5. 2人目以降のユーザーは、**新規登録**で店舗を選んで登録すると**キャスト**として作成されます。管理者は**ユーザー管理**でロールや店舗を変更できます。

---

## 5. 起動手順のまとめ（毎回の開発時）

1. **1つ目のターミナル（バックエンド）**
   ```bash
   cd customer_management_system/backend
   source venv/bin/activate   # Windows の場合は venv\Scripts\activate
   python manage.py runserver
   ```

2. **2つ目のターミナル（フロントエンド）**
   ```bash
   cd customer_management_system/frontend
   npm start
   ```

3. ブラウザで **http://localhost:3000** を開き、ログインして利用します。

## 6. よくあるトラブル

| 現象 | 対処例 |
|------|--------|
| フロントから API に接続できない | バックエンドが http://localhost:8000 で起動しているか確認してください。先にバックエンドを起動してから `npm start` してください。 |
| `pip install` でエラーになる | Python のバージョンが 3.10 以上か確認してください。`python-dotenv` でエラーになる場合は、`requirements.txt` のその行を一時的に削除するか、`pip install python-dotenv` で別途インストールを試してください。 |
| データベース接続エラー | `settings.py` のデータベース名・ユーザー名・パスワードが、PostgreSQL の実際の設定と一致しているか確認してください。 |
| ログインできない | 初回は**新規登録**でアカウントを作成してから、ログインしてください。 |
| スマートフォンでメニューが出ない | 画面幅が狭いときは、画面上部の**ハンバーガーメニュー（≡）**をタップするとメニューが開きます。 |
